# Stakeholders & Users

## Target Audience

| Persona   | Description                                                           | Key Needs                                                                                                                                           |
| --------- | --------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| Developer | Team developers and future contributors working on Unity game project | Modular, reusable components, documented API, ease of integration between client and backend, reference implementations for gameplay patterns (FSM) |
| Game User | End users playing the cross-platform game via Unity interface         | Stable gameplay, intuitive UI, consistent game logic, reliable feature interactions                                                                 |

## User Personas

### Developer

| Attribute      | Details                                                                                                                |
| -------------- | ---------------------------------------------------------------------------------------------------------------------- |
| Role           | Student/Unity developer                                                                                                |
| Age            | 16+                                                                                                                    |
| Tech Savviness | High                                                                                                                   |
| Goals          | Implement and extend game mechanics, AI, backend; maintain and scale project                                           |
| Frustrations   | Tightly coupled scripts, incomplete backend integration                                                                |
| Scenario       | Adds new enemy types or features using existing modules, integrates with API, uses persisted data for future balancing |

### Game User

| Attribute      | Details                                                                                                            |
| -------------- | ------------------------------------------------------------------------------------------------------------------ |
| Role           | Player                                                                                                             |
| Age            | 10+                                                                                                                |
| Tech Savviness | Low–Medium                                                                                                         |
| Goals          | Enjoy stable and engaging gameplay; save progress                                                                  |
| Frustrations   | Errors in enemy behavior, lost progress if app deleted                                                             |
| Scenario       | Plays cross-platform desktop game, interacts via GUI, earns achievements, tracks stats without technical knowledge |

## Stakeholder Map

**High Influence / High Interest:** Developer – implements, tests, documents technical components, ensures module integration

**High Influence / Low Interest:** Experts/Commission – evaluate compliance and documentation; periodic engagement

**Low Influence / High Interest:** Game development team (designers, animators, sound, narrative) – relies on stable technical system to create content

**Low Influence / Low Interest:** Scientific Supervisor – provides guidance; potential users – interested in stable gameplay and progress saving
